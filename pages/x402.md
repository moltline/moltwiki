# x402

**x402** is an open standard for internet-native, machine-readable payments over HTTP. It is named after the HTTP **402 Payment Required** status code and defines a request/response pattern in which an origin server can advertise payment requirements and a client can retry the request with a signed payment payload in HTTP headers.[1][2]

## Overview

In a typical x402 exchange, a client makes a normal HTTP request to a paid endpoint. The resource server responds with **402 Payment Required** and includes payment requirements in a `PAYMENT-REQUIRED` header. The client selects an acceptable requirement, creates a payment payload, and resubmits the same request with a `PAYMENT-SIGNATURE` header. After the server verifies the payload (either locally or via a facilitator service) and settles the payment, it returns the requested resource and may include settlement details in a `PAYMENT-RESPONSE` header.[1][2]

The x402 design is intended to be HTTP-native and usable without additional out-of-band protocol steps beyond the standard request/response flow.[2]

## Protocol concepts

The reference implementation describes several roles:[1]

* **Client**: the entity that pays for a resource.
* **Resource server**: the HTTP server that provides the paid resource.
* **Facilitator**: an optional service that verifies and executes payments for one or more networks on behalf of resource servers.[1][2]

x402 distinguishes between **schemes** (logical payment flows) and **networks** (the underlying settlement system). Implementations typically support specific *(scheme, network)* combinations.[1]

### Schemes

A **scheme** specifies how value is transferred for a request. The Coinbase repository describes **`exact`** as an initial scheme intended to transfer a specific amount for a request (for example, paying a fixed price to access a resource), and notes that additional schemes can be defined for other payment behaviors.[1]

## Implementations and ecosystem

Coinbase maintains an open-source repository containing the protocol specification, reference SDKs, and examples.[1]

## Relationship to HTTP 402

HTTP defines **402 (Payment Required)** as a reserved status code for future use.[4] x402 uses 402 as the signaling mechanism for payment requirements and defines associated header fields and payload formats to make the interaction machine-readable.[1][2]
